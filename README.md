# Beyond-Accuracy — Simulation-Based Safety Evaluation for Decision-Support Classifiers (Beyond-Accuracy)

> Companion code that recomputes, from a single fixed seed, the safety-behaviour metrics and the deployment verdict reported for the *Beyond Accuracy* study.

![License](https://img.shields.io/badge/license-MIT-blue) ![Python](https://img.shields.io/badge/python-3.10%2B-blue) ![Reproducible](https://img.shields.io/badge/reproducible-seed--42-success)

## Overview

A classifier can post a high held-out accuracy and still behave badly the moment its inputs degrade or the cost of a mistake stops being uniform. Picking models on average-case correctness leaves those behaviours unmeasured, which is exactly where decision support tends to fail after it ships. This work treats pre-deployment evaluation as a stress test rather than a leaderboard: it asks whether a model stays calibrated across urgency strata, whether it abstains sensibly as inputs worsen, whether its errors pile up in the costly cases, and whether its explanations hold steady under small perturbations.

The accompanying paper formalises that idea as a five-element framework — input archetypes, perturbation operators, a scenario builder, a safety-metric suite, and a verdict function — backed by two formal results (a completeness argument and a finite-sample conformal-coverage bound). This repository is the executable half. It builds a deterministic synthetic cohort, fits a calibrated classifier, and computes the five-metric suite together with the governance verdict, so a reader can rerun the pipeline end to end and inspect every number rather than take the manuscript on trust.

We keep one boundary explicit. The numbers produced here come from a self-contained synthetic generator wired to public digital-twin disease models; they exercise the metric machinery and the verdict logic honestly, but they are not the manuscript's tuned dizziness-support prototype. Where the two diverge, this README reports the repository value and flags the gap (see *Key results* and *Relationship to the manuscript*).

## Key results

All values below are read directly from `results/eval_results.json`, regenerated by `run_all.py` at seed 42 on a synthetic, in-distribution cohort.

- Cohort of 501 scenarios (167 per urgency tier), 48 summary features, split 250 / 125 / 126 for train / calibration / test.
- Test accuracy 0.810 and AUROC 0.925 — discrimination is solid, which is precisely why accuracy alone hides the safety picture.
- Calibration error climbs with urgency: ECE 0.083 (Low) → 0.135 (Medium) → 0.163 (High); overall 0.080. The coverage-risk area moves the same way: AURC 0.101 → 0.116 → 0.223; overall 0.146.
- Tier-aware thresholding cuts harm-weighted loss from 1.159 to 0.992, a 14.4% reduction, without retraining the model.
- Explanation-ranking stability under perturbation is high (Spearman 0.975), and split-conformal coverage reaches 0.929 against a 0.90 target.
- The verdict function returns **Conditional (G = 0.60)**, failing two of five gates — high-tier calibration and harm-weighted loss — which become the named remediation targets.

The headline verdict matches the manuscript (Conditional, G = 0.60) and the central qualitative claim holds: calibration and coverage-risk both worsen monotonically from Low to High tier. The point-estimate magnitudes differ from the manuscript because the cohort, the model family, and the harm-weight scale differ; those differences are itemised below rather than smoothed over.

## Reproducing the results

```bash
python run_all.py          # fixed seed 42; a few minutes on a standard workstation
```

The driver seeds NumPy globally and passes `random_state=42` into every split and estimator, so the run is deterministic: the cohort, the calibrated random forest, the SHAP perturbation draw, and the conformal split are all fixed. Rerunning on the same library versions reproduces the metrics in `results/eval_results.json` exactly — no value is hardcoded, each is computed from the seed at run time. The one component with an external dependency is the SHAP step (`explanation_stability`), which needs the `shap` package; everything else relies only on NumPy, pandas, scikit-learn, and SciPy.

Outputs written to `results/`:

| File | Contents |
|------|----------|
| `eval_results.json` | Full report: performance, calibration, coverage-risk, harm, explanation stability, conformal coverage, governance verdict |
| `safety_metrics.csv` | The five-metric safety suite (overall) |
| `metrics_by_tier.csv` | Per-tier ECE / AURC / harm breakdown |
| `governance_gates.csv` | Five-gate verdict outcome and score |

## Results and figures

The repository ships computed metrics plus the figure *source* scripts rather than rendered binaries: `figures/*.py` draw the schematic diagrams (conceptual framework, pipelines, lifecycle) and the data panels (coverage–risk curve, tier-stratified harm, calibration/XAI). The data panels and the graphical abstract read their numbers at run time from `results/eval_results.json`, so they always track `run_all.py`'s computed outputs — nothing is hardcoded, and the scripts write into `figures/` (not an external path). The rendered outputs are git-ignored; regenerate them with `python figures/generate_figures.py` (run `python run_all.py` first). The numbers a reader should rely on live in `results/`; the findings below reference those computed CSV/JSON outputs.

- `results/metrics_by_tier.csv` — calibration error and coverage-risk area broken out by urgency tier. Read down the High-tier row: ECE 0.163 and AURC 0.223 are the largest in their columns, which is the framework's core signal that the model is least trustworthy exactly where mistakes cost the most.
- `results/safety_metrics.csv` — the five-metric suite at a glance (ECE 0.080, AURC 0.146, harm-weighted loss 0.992, explanation stability 0.975, conformal coverage 0.929). The contrast between strong discrimination and a Conditional verdict is the take-away: aggregate scores look fine while two safety gates fail.
- `results/governance_gates.csv` — the verdict table. Three gates pass (coverage-risk, conformal coverage, explanation stability) and two fail (high-tier calibration, harm-weighted loss), giving G = 0.60 and a Conditional disposition with explicit remediation targets.
- `results/eval_results.json` — the complete record, including the harm breakdown that shows the tier-aware policy cutting High-tier harm from 2.857 to 2.381 while leaving Low and Medium essentially unchanged.

To redraw the figures locally, run `python run_all.py` (writes `results/`) then `python figures/generate_figures.py` (and, optionally, `figures/graphical_abstract.py` and `figures/generate_figures_drawio.py`). The data panels render from `results/eval_results.json`, so they reflect the current run.

## Data

No human-subjects data are used and no ethics approval is required. The cohort is synthetic: each patient trajectory is rolled forward through one of three digital-twin disease models (Sepsis, respiratory distress, cardiac event), summarised into last / mean / max / slope features, and labelled by passing a standardised latent-severity signal through a logistic link and sampling. Label noise rises with urgency tier, so the high-acuity cases are deliberately the most ambiguous — the pattern the safety metrics are built to expose. Every generator parameter is a fixed, documented constant applied under seed 42; none is tuned toward a target number.

## Relationship to the manuscript

The repository run and the manuscript prototype share the framework, the metric definitions, the verdict logic, and the headline verdict (Conditional, G = 0.60), but they use a different cohort, model, and harm scale. The differences are intentional and listed here so no reader mistakes one for the other:

| Quantity | This repository | Manuscript prototype |
|---|---|---|
| Cohort size | 501 scenarios | 500 scenarios |
| Features | 48 | 18 |
| Classifier | calibrated random forest | gradient-boosted trees |
| Accuracy | 0.810 | 0.873 |
| ECE (overall; L / M / H) | 0.080; 0.083 / 0.135 / 0.163 | 0.088; 0.040 / 0.073 / 0.131 |
| AURC (overall; L / M / H) | 0.146; 0.101 / 0.116 / 0.223 | 0.18; 0.09 / 0.17 / 0.31 |
| Harm-weighted loss (fw vs base) | 0.992 vs 1.159 (−14.4%) | 0.14 vs 0.31 (−54.8%) |
| Explanation stability | 0.975 | 0.81 |
| Conformal coverage (α = 0.10) | 0.929 | 0.913 |
| Verdict (score) | Conditional (0.60) | Conditional (0.60) |

The harm figures differ in scale because the repository weights tiers 1 / 3 / 10 while the manuscript uses 0.10 / 0.30 / 1.00; the qualitative effect — a tier-aware threshold lowering harm without retraining — is the same in both.
